# Анализ требований и проектирование тестов для мобильного приложения «Найди и иди»

## 1. Исходные данные (Техническое задание)

Приложение «Найди и иди» предназначено для поиска нужного пользователям магазина и построения маршрута до него. Для работы системы геолокация всегда разрешена приложению.

## 2. Анализ и тестирование требований


### SC21-01 Поле ввода названия магазина
**Фраза:** «Поле ввода названия магазина принимает любые символы»  
    - Критерий: Полнота.  
    - Пояснение: Не указано ограничение на максимальную длину строки (граничные значения). Также отсутствует валидация на ввод строки, состоящей только из пробелов.  
**Фраза:** «В поле сохраняются результаты предыдущих поисков»  
    - Критерий: Однозначность / Полнота.  
    - Пояснение: Не регламентировано, как именно отображается история (выпадающий список, автозаполнение). Не указано количество хранимых записей и возможность очистки истории пользователем.  

### SC21-02 Поле выбора типа магазина
**Фраза:** «и т.п.»  
    - Критерий: Полнота.  
    - Пояснение: Окончательный список значений не завершен. Это создаёт неопределенность для разработки и делает невозможным точное написание тест-кейсов.  
**Фраза:** «При наличии выбранного значения оно отображается в поле»  
    - Критерий: Однозначность.  
    - Пояснение: Не описана возможность сбросить или очистить выбранный тип магазина, чтобы вернуть поле в исходное состояние.  

### SC21-03 Кнопка «Поиск»
**Фраза:** «Успешные результаты поиска отображаются в виде флажков на карте»  
    - Критерий: Полнота.  
    - Пояснение: Не указано поведение системы в негативном сценарии — когда результаты поиска отсутствуют (нет текста ошибки или специального UI-уведомления). Также не зафиксирован внешний вид флажков.  
**Фраза:** «Карту можно скроллить и зуммировать (стандартные жесты)»  
    - Критерий: Однозначность.  
    - Пояснение: Формулировка «стандартные жесты» субъективна. Необходимо четко прописать жесты для мобильных устройств (например, скролл одним пальцем, зум двумя пальцами).  

### SC21-04 Кнопка «Построить маршрут»
**Фраза:** «Найден результат поиска введенного пользователем названия магазина и выбран его флаг на карте»  
    - Критерий: Однозначность.  
    - Пояснение: Не указано, какое действие означает «выбран» (одиночное нажатие, двойной тап). Не описано визуальное состояние выбранного флага (подсветка, изменение цвета или размера).  
**Фраза:** «при нажатии на кнопку цвет фона меняется на #586d73 (серый) и название кнопки меняется с «Построить маршрут» на «Отменить маршрут»»  
    - Критерий: Атомарность.  
    - Пояснение: Требование описывает одновременно два разных изменения (визуальное состояние и текст). Их следует разделить на отдельные подпункты.  

### SC21-05 Область карты
**Фраза:** «Область карты занимает половину области экрана телефона»  
    - Критерий: Однозначность.  
    - Пояснение: Не указано, какую конкретно половину экрана (верхнюю или нижнюю) занимает карта, а также как меняется интерфейс при смене ориентации устройства (портретный/альбомный режим).  
**Фраза:** «Построенный маршрут можно отменить только нажатием кнопки «Отмена»»  
    - Критерий: Непересекаемость (Противоречие).  
    - Пояснение: Прямой конфликт с требованием **SC21-04**, согласно которому кнопка меняет текст на «Отменить маршрут». Из ТЗ неясно, это одна и та же динамическая кнопка или в интерфейсе появляется отдельный элемент «Отмена».

### SC21-06 Общие требования к работе приложения
**Фраза:** «При повторном открытии приложения отображать незаконченный ранее маршрут»  
    - Критерий: Полнота.  
    - Пояснение: Не определены критерии «незаконченного» маршрута (пользователь не дошел до цели, или приложение просто было свернуто) и время хранения этой сессии.  
**Фраза:** «Отключать точную геопозицию при низком заряде аккумулятора»  
    - Критерий: Однозначность.  
    - Пояснение: Не указано конкретное числовое значение «низкого заряда» в процентах (например, 15% или 20%) и не описано, уведомляется ли об этом пользователь.

## 3. Чек-лист проверок (Black-box)

### Поле ввода названия магазина (SC21-01)
[ ] Отображение поля ввода названия магазина на экране.  
[ ] Отображение корректного текста плейсхолдера по умолчанию.  
[ ] Плейсхолдер полностью исчезает сразу при вводе первого символа.  
[ ] Плейсхолдер возвращается, если полностью стереть введенный текст.  
[ ] Поле корректно принимает различные типы символов (буквы, цифры, спецсимволы).  
[ ] Ввод менее 2 символов блокирует возможность выполнения поиска.  
[ ] Ввод 2 и более символов делает функцию поиска доступной.  
[ ] Введенное значение успешно сохраняется в истории приложения после выполнения поиска.  
[ ] Предыдущие поисковые запросы отображаются в виде выпадающего списка при повторном фокусе на поле.

### Поле выбора типа магазина (SC21-02)
[ ] Отображение поля выбора типа магазина на экране.  
[ ] Отображение плейсхолдера «Выберите тип магазина» по умолчанию.  
[ ] Открытие выпадающего меню при нажатии на поле.  
[ ] Наличие в выпадающем списке обязательных вариантов («Продуктовые магазины», «Магазины стройматериалов», «Аптеки»).  
[ ] Возможность выбрать только одно значение из списка за один раз.  
[ ] Выбранное значение корректно отображается в поле, поле считается заполненным.

### Кнопка «Поиск» (SC21-03)
[ ] Отображение кнопки «Поиск» на экране.  
[ ] Кнопка находится в неактивном состоянии (серая/некликабельная), если оба поля ввода пусты.  
[ ] Кнопка активируется, если заполнено только поле ввода названия магазина.  
[ ] Кнопка активируется, если заполнен только тип магазина.  
[ ] Нажатие на активную кнопку запускает процесс поиска.  
[ ] Успешные результаты поиска визуализируются на карте в виде флажков.  
[ ] Появление информационного сообщения (например, «Ничего не найдено»), если результаты отсутствуют.  
[ ] Возможность перемещать (скроллить) карту свайпом одного пальца.  
[ ] Возможность изменять масштаб карты (зум) стандартным жестом сведения/разведения двух пальцев.

### Кнопка «Построить маршрут» (SC21-04)
[ ] Отображение кнопки на экране, проверка UI-элементов (иконка в виде изогнутой стрелки, надпись «Построить маршрут», белый фон `#fff`).  
[ ] Кнопка неактивна, если условия для построения маршрута не выполнены.  
[ ] Кнопка активируется при успешном поиске по названию и последующем выборе флажка магазина на карте.  
[ ] Кнопка активируется сразу после выбора типа магазина в выпадающем списке (маршрут до ближайшего).  
[ ] Нажатие на кнопку инициирует построение и отрисовку маршрута на карте.  
[ ] Сразу после нажатия текст кнопки меняется на «Отменить маршрут».  
[ ] Сразу после нажатия цвет фона кнопки меняется на серый (`#586d73`).

### Отмена маршрута и область карты (SC21-04 + SC21-05)
[ ] Отображение карты ровно на 50% от высоты области экрана телефона.  
[ ] Нажатие на кнопку «Отменить маршрут» полностью удаляет текущий маршрут с карты.  
[ ] После отмены маршрута кнопка возвращается в исходное состояние (текст «Построить маршрут», фон `#fff`).  
[ ] Проверка невозможности отменить маршрут другими путями (например, случайным тапом по свободной области карты).

### Общие проверки работы приложения (SC21-06)
[ ] Проверка сохранения сессии: при закрытии и повторном открытии приложения незавершенный маршрут автоматически восстанавливается и отображается на карте.  
[ ] Непрерывное использование приложением геолокации во время активного процесса работы.  
[ ] Отключение точной геопозиции (переход в энергосберегающий режим) при симуляции низкого заряда батареи устройства (например, <= 20%).  
[ ] Стабильность работы приложения и динамическое обновление данных при изменении текущих координат устройства.
